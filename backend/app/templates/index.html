System Architecture

End-to-end RAG workflow

01

Upload PDF

Accepts research papers through the backend API.

02

Extract Text

Uses PyMuPDF to extract page-level text from PDFs.

03

Chunk Text

Splits documents into overlapping chunks for retrieval.

04

Embed Chunks

Converts text chunks into dense vector embeddings.

05

Store Vectors

Persists embeddings and metadata in ChromaDB.

06

Retrieve Context

Finds the most relevant chunks for a user question.

07

Generate Answer

Builds a grounded response from retrieved chunks.

08

Return Citations

Shows source labels with filename and chunk ID.

API Layer

Available backend endpoints

GET /health

Health Check

Checks whether the CiteMind backend service is running.

POST /upload-pdf

Upload and Index PDF

Uploads a paper, extracts text, chunks it, embeds it, and stores it in ChromaDB.

POST /search

Similarity Search

Retrieves the most relevant document chunks for a question.

POST /ask

Ask with Citations

Returns a grounded answer with clean source citations.

GET /vector-store/stats

Vector Store Stats

Displays collection name and total stored chunks.
